The NOVAK Execution Flow: API Integration

How an existing Microservice attaches the Integrity Layer
1
Microservice Execution
The enterprise application (e.g., a credit scoring service) runs its business logic as normal.
↓
2
Gather NIPS Data (Input + Output)
The service collects all **verifiable inputs** (data, user ID, previous state) and **verifiable outputs** (the decision result).
NIPS_PAYLOAD = { rule_id: "VA_2025.LOAN_APPROVAL", input: {...}, output: "DENY" }
↓
3
Call NOVAK Protocol API
The service makes a single, non-blocking call to the locally deployed NOVAK engine, passing the NIPS data and the **Previous REVELATION Hash**.
POST /v1/novak/attest → (NIPS_PAYLOAD, PREV_CHAIN_HASH)
↓
4
HARMONEE Receipt & REVELATION Chain
The NOVAK engine cryptographically hashes the input (creating **HARMONEE**) and links it to the global chain (creating the new **REVELATION** hash).
↓
5
Store & Proceed
The service receives the **REVELATION Hash** and saves it alongside its traditional database record, completing the verifiable transaction.
Database Record: {..., final_decision: "DENY", **revelation_hash: "0x4b7c..."**}
The application continues to run quickly, but now **every single decision** is automatically and instantly locked into the immutable **REVELATION Chain**.
